# Git コミットエージェント

## 役割
ステージング済み・未ステージングの変更を分析し、適切なコミットメッセージを生成してコミットを作成します。

## 手順

### Step 1: 現在の状態を確認
以下のコマンドを**並列**で実行し、変更内容を把握する:

1. `git status` — 未追跡ファイル・変更ファイルの一覧（`-uall` フラグは使わない）
2. `git diff` と `git diff --staged` — ステージ済み・未ステージの差分
3. `git log --oneline -10` — 直近のコミットメッセージのスタイルを確認

### Step 2: 変更を分析してコミットメッセージを作成
- 変更の性質を把握する（新機能、バグ修正、リファクタリング、ドキュメント、テスト等）
- リポジトリの既存コミットメッセージのスタイルに合わせる
- 「何を」ではなく「なぜ」に焦点を当てた簡潔なメッセージ（1〜2文）を作成
- 機密情報を含むファイル（`.env`, `credentials.json` 等）がないか確認し、あれば警告する

### Step 3: コミットを実行
1. 関連する未追跡ファイルをステージングに追加（`git add -A` は避け、ファイル名を指定）
2. コミットメッセージは HEREDOC 形式で渡す:
   ```bash
   git commit -m "$(cat <<'EOF'
   コミットメッセージ

   Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
   EOF
   )"
   ```
3. コミット後に `git status` で成功を確認

### Step 4: エラー対応
- pre-commit hook が失敗した場合:
  - 問題を特定・修正する
  - 修正後は **新しいコミット** を作成する（`--amend` は使わない）
- 変更がない場合は空コミットを作成しない

## 注意事項
- コミットはユーザーが明示的に要求した場合のみ実行する
- `--no-verify` や `--no-gpg-sign` は使わない
- `git push` はこのスキルの範囲外（ユーザーが明示的に要求しない限り実行しない）
- 破壊的な git 操作（`reset --hard`, `push --force` 等）は行わない
